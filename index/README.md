# index/ — 적재기(2단계) 산출물

`scripts/build_index.py`가 `corpus_md/`(102파일·9,885문단) + `guidelines_md/`(9파일·177문단)를
Qdrant Cloud 컬렉션 `standards_20250829_bgem3`으로 적재하며 생성한다.
지시서는 `TASK.md`(v2 — Qdrant판), 참조 규약은 `guidelines_raw/벡터저장소_스키마_및_마크다운_작성규약.md`(v2.1).

## 산출물 (커밋 대상)

| 파일 | 내용 |
|---|---|
| `vocab.json` | sparse 어휘집(토큰→정수 인덱스) + 메타(k1·b·avgdl·kiwipiepy 버전·품사 규칙) |
| `glossary.jsonl` | 용어 사전 — 원천 ① 정의 조각 280 + ② 구기준 기울임-콜론 + ③ 감사기준 정의 문단 |
| `manifest.json` | 재구축 계약서: 코퍼스 커밋 해시·컬렉션·UUID5 네임스페이스·임베딩/스파스 파라미터. **3단계 검색기는 기동 시 이 파일을 읽어 모델 동일성 불일치면 기동 실패해야 한다** |
| `build_report.md` | 최근 실행의 점검 1~7·스모크 S1~S9·P1~P3 결과 |
| `cache/` | dense 임베딩 캐시(.npy) — gitignore, 소모품 |

재구축: `.venv/bin/python scripts/build_index.py` (접속 정보는 `.env`의 `QDRANT_URL`/`QDRANT_API_KEY` — 커밋 금지).

### dense 임베딩의 Colab GPU 이관 경로

CPU 임베딩(1~2시간)을 피하기 위해 dense만 외부 GPU로 계산할 수 있다:

1. `--stage export` → `index/embed_input.jsonl` (행당 `{"cid", "text"}`, 10,063행 assert)
2. Colab에서 bge-m3(fp16)로 임베딩 → `embeddings.npy`(float32, L2 정규화)·`cids.json`(npy 행 순서의 cid 목록)·`embed_meta.json`(`embedding_device`/`embedding_runtime`)을 `index/`에 배치
3. `--stage upsert --embeddings index/embeddings.npy --cids index/cids.json`
   — 무결성 검증: 건수 10,063 / cid 집합의 파서 산출 일치(순서가 아니라 **cid 조인**으로 결합) /
   shape (10063,1024)·float32(이탈 시 변환+경고) / L2 노름≈1(이탈 시 재정규화+경고)

`embeddings.npy`·`cids.json`은 gitignore(바이너리). 대신 manifest에
`embeddings_sha256`·`embedding_device`·`embedding_runtime`을 기록한다.
`embed_input.jsonl`은 **커밋 대상으로 전환** — Colab이 저장소 raw에서 입력을 직접 확보하는
워크플로(embedding.ipynb 셀 2)를 위해서다. `--stage export` 재실행으로 언제든 재생성 가능.
Colab(tarball, `.git` 부재) 실행 시 manifest의 코퍼스 커밋은 `CORPUS_COMMIT` 환경변수 폴백으로
기록한다(노트북 셀 2가 GitHub API에서 조회해 설정).

### 이력: FRMK-1·ASSR-3000 standard_title 오염 (2026-07-07 수리)

`normalize_corpus.py`의 보론 진입 분기가 절 제목을 파일 제목과 같은 변수에 재대입해(변수 가림),
마지막 보론 제목이 frontmatter를 덮어썼다 — FRMK-1 "보론 4", ASSR-3000 "보론". 이 상태로 1차
적재가 수행되어 395문단(FRMK-1 109 + ASSR-3000 286)의 합성문 머리말이 오염됐고, 지역 변수 분리로
수리한 뒤 전량 재임베딩·재적재했다. 1차 적재의 점검·스모크는 모두 통과했었으므로(S1~S9는 제목
비의존) 이 결함은 검색 품질에만 영향을 주는 유형이었다.
DB는 소모품 원칙 — 마크다운 원본 + 결정적 UUID5 + 이 폴더의 커밋 산출물만으로 언제든 멱등 재구축된다.

## 구현 확정 사항·이탈 기록

1. **물리/논리 키 분리**: 포인트 ID = `uuid5(uuid5(NAMESPACE_URL, "github.com/worldoftoddl/auditPaper_assist"), 복합ID)`.
   복합 ID는 payload `composite_id`. 왕복 무결성은 스모크 S9가 전수 검사.
2. **문단을 가로지르는 절 제목의 처리** (파서 이탈 아님 — 규칙의 구체화): `## ` 절 행은 절 추적에만 쓰고
   본문에서 제외하되, 절 행 뒤 다음 행머리 전에 일반 텍스트가 이어지는 경우(문단이 여러 절을 가로지르는
   KSA 240 부록1 등)에만 그 절 행의 **말단 세그먼트**를 소제목 텍스트로 본문에 보존한다.
   부수 관찰: 이 규칙은 코퍼스의 문단 경계 계약(다음 행머리 전까지가 한 문단)을 승계하므로,
   kifrs_1109의 `## 정의(부록 A)` 직후 굵은 소제목(`**파생상품**`)이 직전 문단 B7.2.4의 꼬리에
   붙는 사례가 있다 — 코퍼스 단계의 구조이며 적재기에서 재해석하지 않는다.
3. **분할 1건** `KSA::240::부록1` → `#1`(3,846자)·`#2`(1,617자). 절단선 = 최상위 범주 경계인
   "자산의 횡령으로 인한 왜곡표시…" 소제목 행 앞(#1 = 부정한 재무보고 범주까지). 【사람 확인 1】
4. **seq**: 분할 조각 2건은 원문단의 seq를 공유한다(문맥 확장 질의 ⑤에서 두 조각이 같은 자리로
   취급되는 것이 의미상 정확). 연속성 검사는 기준서별 seq 집합 = 1..N으로 수행.
5. **용어 사전 로케이터**: section_path 말단 세그먼트가 정규식 `^(용어의\s*)?정의\s*(\([^)]*\))?$`에
   완전 일치(부분일치 금지). 괄호 변형 실측 3종을 흡수한다 — `용어의 정의(문단 AG3~AG23 참조)`(1032),
   `용어의 정의(문단 11~14)`(1032), `정의(부록 A)`(1109 — 산문 절이라 항목 0건, 무해).
6. **② 구기준 기울임-콜론 두 변형**: `*용어:*`(콜론이 기울임 안 — 1002 등)과 `*용어*:`(콜론이 밖 —
   1001·1007·1016·1019·1021·1023·1024·1026·1028·1032·1034·1040). 정의문 = 다음 용어 마커 전까지.
   관찰: 정의 절 안의 굵은 소제목 행(예: 1007 '현금및현금성자산')이 직전 용어의 정의문 꼬리에
   혼입되는 사례 소수 — 원문 구조 승계.
7. **③ 감사기준 정의 문단 표제 4형식** (표본 확정, 지시서 8장): ⑴ `(x) 용어 – 정의`(대시 변형 –—―- 포함,
   대시가 40자 내), ⑵ `(x) 용어(영문). 정의`(1100), ⑶ 마커 없는 행 단위 `용어 – 정의`
   (210·250·580·701, ASSR-3000의 `- ` 항목 마커 포함), ⑷ 산문형 문단 단위 폴백 —
   `…사용하는(되는) "{용어}"(이)란 …`에서 용어 추출, 정의문 = 문단 전문(320·520).
   KSA::1200::9(보론 1 지시문)는 항목 없음 — 1200의 65용어는 원천 ①로 수록됨.
8. **sparse 토큰 규칙**: kiwipiepy 형태소 중 NNG·NNP·XR(명사·어근)·SL(영문, 소문자화)·SN(숫자)·SH(한자)만
   유지. 문서측 BM25 가중(k1=1.5, b=0.75)을 클라이언트가 선적용, 질의측은 토큰당 1.0(IDF는 Qdrant
   서버의 IDF modifier). **질의 시 어휘집 밖 토큰은 무시된다** — 어휘집이 코퍼스 전수에서 구축되므로
   코퍼스에 없는 어휘는 sparse 경로에서 매칭 대상 자체가 없어 정상 동작이다.
9. **임베딩**: `BAAI/bge-m3` dense 1024차원, normalize=True, sentence-transformers, 전 문단
   합성문(`{유형} {번호} {제목} | {절 경로} | {본문}`) 전문 임베딩. 8192토큰 초과는 절단하지 않고
   중단·기록(현 코퍼스 0건). 표 요지·장문 추가 분할은 측정 게이트(P1~P3) 판단으로 이관.
