# 적재기(2단계) 구현 지시서 v2 — Qdrant판 (전문)

이 문서는 v1(Chroma판)을 **대체**하는 자기완결 지시서다. 저장소를 Qdrant Cloud로 확정함에 따른 개정이며, 참조 규약은 `guidelines_raw/벡터저장소_스키마_및_마크다운_작성규약.md`(v2.1)로 불변이다 — 이번 전환은 규약 3.5에 예약된 이식성 조항의 행사이지 논리 스키마 변경이 아니다. 근거 있는 구현 이탈은 README 기록을 조건으로 허용한다.

**v1 대비 변경 지도** — ① 수치 갱신(0·2장: 10,062문단), ② UUID5 물리 키 + 왕복 검증(5장·S9), ③ sparse 벡터 통합·클라이언트 가중·vocab.json 커밋(7장), ④ 모델 태그 컬렉션 명명(5장), ⑤ 파생물 커밋 원칙 유지(11장). 신설: 13장 KURE 벤치마크, 14장 호스팅 계약. **불변**: 2~4장(파서·유도·합성), 6장(분할), 8장(용어 사전), 9장(점검), 12장(측정 게이트).

## 0. 입력·전제 (확정 상태)

- **입력 코퍼스**: 커밋 `b53a44e` — `corpus_md/` 102개 파일·9,885문단(감사기준 3,630 + 회계기준 6,255) + `guidelines_md/` 9개 파일·177문단 = **111개 파일·10,062문단**. 결함 0, 검증기 통과.
- **저장소**: Qdrant Cloud 무료 티어. 규모 근거: 10,062문단 × 1024차원 × 4바이트 ≈ 41MB(dense) — A/B용 제2 컬렉션을 더해도 1GB 한도 내 여유.
- **6,000자 초과**: `KSA::240::부록1`(6,024자) 1건 — 유일한 필수 분할 대상(6장).
- **행머리 정규식**: `scripts/normalize_corpus.py`의 `HEAD_RE`를 import하여 공유(복제 금지). 유형코드: 감사기준→`KSA`, 회계기준→`KIFRS`, 실무지침→`GUIDE`.
- **핵심 원칙 (신설)**: **물리 키와 논리 식별자의 분리.** Qdrant 포인트 ID는 u64/UUID만 허용하므로 물리 키는 UUID5(결정적)로 두고, 우리의 복합 ID(`유형코드::기준서번호::문단번호[#순번]`)는 payload에 논리 식별자로 보관한다. 규약 3.6 "id는 주소, 메타데이터는 사실"의 연장이며, 인용·라우팅·검증은 전부 논리 식별자 기준으로 동작한다.

## 1. 구성요소 지도

```
corpus_md/·guidelines_md/ ─▶ [A 파서] ─▶ 문단 레코드
                                │
                     [B para_type 유도] ─▶ [C 분할: 240::부록1만]
                                │
                ┌───────────────┴───────────────┐
        [D dense 임베딩 합성·계산]        [E sparse 벡터 생성]
                └───────────────┬───────────────┘
                        [G Qdrant 업서트: named vectors]
                                │
        [F 용어 사전 파생]   [H 점검 1~7 + 스모크 S1~S9] ─▶ 매니페스트
```

v1의 "별도 BM25 색인"은 사라지고, dense·sparse가 **한 컬렉션의 named vector 두 개**로 통합된다. 하이브리드 융합(RRF)은 서버 측 Query API가 수행하므로 3단계의 융합 코드가 통째로 불필요해진다.

## 2. [A] 파서 사양 (v1과 동일, 상수만 확정)

1. frontmatter 3필드(`source_type`·`standard_no` 따옴표 제거·`standard_title`), 그 외 필드 무시.
2. `## ` 행으로 절 추적 — 문단의 `section_path` = 행머리 시점의 절 제목. 절 제목 행은 본문에 불포함.
3. `HEAD_RE` 절단, `document` = 행머리 제거 본문 전체(표·각주·굵은체 마커 포함 — 원문 충실).
4. 산출 필드: 복합 ID, `para_no`, `seq`(기준서 내 1부터), `has_table`, `char_len`, `source_file`.
5. **총계 고정 검증**: 파싱 직후 assert — 감사기준 **3,630** / 회계기준 **6,255** / 실무지침 **177** (계 10,062). 코퍼스 갱신 시 의식적 갱신을 강제하는 드리프트 방지 장치.

접두 규칙을 위에서부터 순서대로 적용한다(공통 규칙 우선).

| 우선 | 조건 | para_type | 기대 규모 |
|---|---|---|---|
| 1 | `정의-` 접두 | 정의 | 280 |
| 2 | `참조N` | 참조 | 14 |
| 3 | `부록N` · `부록-사례N` · `사례X` | 부록 | 예시류 |
| 4 | 감사기준: 순수 숫자 | 요구사항 | |
| 5 | 감사기준: `A`+숫자 / `부록-N` / `보론N-M` | 적용지침 | |
| 6 | 회계기준: 순수 숫자 · `한`접두 | 본문 | |
| 7 | 회계기준: 알파벳 접두(A/B/C/D/E/BA/AG 등) | 적용지침 | |
| 8 | 실무지침: 숫자 | 본문 | |

산출 분포를 점검 리포트에 출력하고 정의 280·참조 14는 기대값으로 고정한다.

## 4. [D] document·dense 임베딩 합성 (v1과 동일)

- 저장(document) = 깨끗한 원문. 합성문 = `{source_type} {standard_no} {standard_title} | {section_path} | {document}`. 분할 조각은 section_path 뒤 ` (n/m)`, 예시류는 ` · 부록 예시` 표지.
- 모델: `BAAI/bge-m3` dense(1024차원), `normalize_embeddings=True`, 최대 8192토큰, 배치 16~32. **v1 원칙 유지**: 전 문단 합성문 전문 임베딩 — 표 요지·장문 분할은 측정 게이트(12장).
- 8192토큰 초과 발생 시 절단 금지, 실패 목록 기록 후 사람 판단.

## 5. [G] Qdrant 적재 사양 (전면 개정)

1. **컬렉션 명명**: `standards_20250829_bgem3` — 규약 3.1의 "판본은 컬렉션 이름" 원칙을 **기준선 날짜 + 임베딩 모델 태그**로 확장한다(임베딩 모델도 벡터의 의미를 결정하므로). KURE A/B 시 `standards_20250829_kure1` 별도 생성(13장).
2. **벡터 구성**: named vectors — `dense`(size 1024, distance Cosine), `sparse`(SparseVectorParams, **IDF modifier**).
3. **물리 키**: `PROJECT_NS = uuid5(NAMESPACE_URL, "github.com/worldoftoddl/auditPaper_assist")`, `point_id = uuid5(PROJECT_NS, 복합ID)`. 완전 결정적이므로 멱등 업서트가 유지된다. 네임스페이스 문자열은 매니페스트에 기록.
4. **payload 스키마**: `composite_id`(복합 ID 원문), `source_type`, `standard_no`, `standard_title`, `para_no`, `para_type`, `section_path`, `seq`, `has_table`, `char_len`, `source_file`, (분할 레코드만) `part_no`·`part_total`. `cross_refs`는 v2 백로그 — 이번에 넣지 않는다.
5. **payload 인덱스**: keyword — `composite_id`, `source_type`, `standard_no`, `para_no`, `para_type` / integer — `seq`(질의 ⑤ 범위 조회용), `part_no`. 결정론적 라우팅(질의 ①)과 인용 대조 폴백(④), 문맥 확장(⑤)이 전부 이 인덱스로 동작한다.
6. **인용 대조 2단계(④)의 Qdrant 구현**: 인용된 복합 ID → UUID5 재계산 → 포인트 직조회 → 실패 시 payload 등가 조회(`standard_no`+`para_no`; 분할 조각 `부록1#N`은 기존 규칙대로 `#` 앞부분으로 조회).
7. 배치 업서트(예: 256포인트), 재실행 시 결과 동일. 적재 전 복합 ID 중복 사전 검사(발견 즉시 실패).
8. **접속 정보 취급**: 클러스터 URL·API 키는 환경변수(`QDRANT_URL`, `QDRANT_API_KEY`)로만 — 코드·매니페스트·커밋에 절대 포함 금지. `.env`는 gitignore.

## 6. [C] 필수 분할 1건 — `KSA::240::부록1` (v1과 동일)

원문의 최상위 범주 경계(부정한 재무보고 / 자산의 보호)에서 2조각. 논리 ID `KSA::240::부록1#1`·`#2`(물리 키는 각각의 UUID5), `para_no`는 `부록1` 유지, `part_no` 1·2, `part_total` 2, 합성문 `(1/2)`·`(2/2)`. 절단 행은 README 기록(사람 확인). 재조립 규칙(3단계): 어느 조각이 걸리든 `standard_no`+`para_no` 등가 조회로 전 조각을 원순서 투입.

## 7. [E] sparse 벡터 사양 (개정 — 별도 색인 대체)

1. **토큰화**: kiwipiepy 형태소 분석(명사·어근 중심, 숫자·영문 보존). 대상 텍스트 = 4장의 합성문(머리말 포함 — 기준서명 일치를 공짜로 얻음).
2. **어휘집**: 코퍼스 전체 토큰 → 정수 인덱스 매핑을 구축해 `index/vocab.json`으로 **커밋**한다(v1의 bm25.pkl을 대체 — diff 가능한 텍스트라 오히려 개선). 질의 시 어휘집 밖 토큰은 무시(코퍼스 전용어이므로 정상 동작이며, 그 사실을 README에 한 줄).
3. **가중 규칙 — 중요**: Qdrant의 IDF modifier는 역문서빈도만 서버가 적용한다. BM25의 나머지 절반은 **문서 측 값에 클라이언트가 미리 가중**해야 한다:
   `문서측 값 = tf × (k1+1) / (tf + k1 × (1 − b + b × dl/avgdl))`, 기본 k1=1.5, b=0.75, dl=문서 토큰 수, avgdl=코퍼스 평균. **질의측 값 = 토큰당 1.0**(IDF는 서버가 질의에 적용). k1·b·avgdl·어휘집 크기·kiwipiepy 버전을 vocab.json 메타에 기록.
4. 전 포인트가 dense·sparse 두 벡터를 모두 보유해야 한다(점검 9장 5행).

## 8. [F] 용어 사전 파생 (v1과 동일)

세 원천 — ① para_type="정의" 280건, ② 구기준: section_path **말단이** '정의'/'용어의 정의'인 문단의 기울임-콜론 표기 `*{용어}:*` 파싱(부분일치 금지 — 계층 경로의 말단 세그먼트만), ③ 감사기준: 같은 로케이터, 표제 형식은 표본 2~3개로 확정 후 README 기록(파싱 불가 시 문단 단위 폴백). 산출 `index/glossary.jsonl`(**커밋**): `{"term", "term_norm", "source_id"(복합 ID), "definition", "standard_no", "source_type"}`. 동일 용어의 복수 정의는 모두 보존(해소는 3단계: 문맥 기준서 우선 → 정의 보유 기준서). 커버리지: 정의 280 일치 + 구기준 파일별 수 리포트 + 표본 3파일 원문 대조(사람 확인). **주입 실행 로직은 3단계 소관.**

## 9. [H] 적재 후 점검 자동화 (규약 5장 대응)

| 규약 | 적재기 구현 |
|---|---|
| 1 문단 수 | 파싱 총계 = 10,062(2장 상수). **적재 포인트 수 = 10,063**(분할로 부록1 1건 → 2조각, 순증 +1) — 이 정합식을 assert |
| 2 연속성 | 원문 번호 연속성은 코퍼스 검증기 승계(허용목록 14곳). 적재기는 `seq` 1..N 연속 검사 |
| 3 id 중복 | 복합 ID 사전 중복 검사 0건 + UUID5 충돌 검사(이론상 불가하나 assert) + 컬렉션 count 대조 |
| 4 표 표본 | `has_table` 상위 5건 복합 ID 리포트 출력 → 사람 육안 |
| 5 벡터 | dense 수 = sparse 수 = 포인트 수, dense 1024차원, NaN·영벡터 0, **sparse 보유율 100%** |
| 6 무마침표 | 코퍼스 검증기 승계 — 매니페스트의 코퍼스 커밋 해시 기록으로 갈음 |
| 7 분할 정합 | part_total 보유 레코드: (standard_no, para_no) 조합이 part_no 1..part_total 결번·중복 없음 — 기대 240::부록1 2조각뿐 |

8번(cross_refs 실재성)은 v2 채움 시점부터 — 리포트에 보류 명시.

## 11. 재현성·산출물·의존성 (개정)

- 스크립트: `scripts/build_index.py` 하나로 파서→적재→사전→점검→스모크 전체 수행, 재실행 멱등.
- 산출물: `index/vocab.json`(**커밋**), `index/glossary.jsonl`(**커밋**), `index/manifest.json`(**커밋**). 매니페스트 기록 항목 — 코퍼스 커밋 해시, 컬렉션 이름(모델 태그 포함), UUID5 네임스페이스 문자열, 임베딩 모델명·리비전·normalize 설정, sparse 파라미터(k1·b·avgdl·어휘집 크기·kiwipiepy 버전), 문단·포인트 수, 생성 시각. **매니페스트가 곧 재구축 계약서이자 질의 측(3단계)이 준수할 모델 동일성 계약서다.**
- 의존성: `qdrant-client`, `FlagEmbedding`(또는 sentence-transformers), `kiwipiepy`. CPU 환경이면 10k 문단 임베딩에 수십 분 소요 가능 — 진행률 출력.
- DB는 소모품 원칙: 마크다운 원본 + 결정적 ID + 위 커밋 산출물만 있으면 언제든 멱등 재구축 가능. 무료 클러스터 휴면 정책은 가입 시 확인(사람 확인 항목).

## 12. 명시적 범위 제외 — 3단계·측정 게이트 이관

이번 구현에 넣지 않는다: 예시류 후순위·옵트인 정책의 실행, 용어 주입 실행, 조서 라우팅 표 연동, 상호참조 역색인(cross_refs v2), 생성 프롬프트. **v1과 달라진 점**: 하이브리드 융합이 서버 측으로 갔으므로 3단계의 융합 코드는 목록에서 삭제 — 3단계는 질의 구성(필터+prefetch)과 정책(옵트인·주입·재조립)만 남는다. 측정 게이트 레버 2개(장문 7건 `#순번` 분할, 표 요지 임베딩)는 P1~P3 결과로 판단.
