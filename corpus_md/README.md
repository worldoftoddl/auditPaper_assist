# corpus_md — 규약 형식 통일 코퍼스 (감사기준 + 회계기준)

`auditstandard_md/`(감사기준 40개)와 `ifrs_md/`+`Conceptual_framework_md/`(K-IFRS 63개)를
`docs/규약_벡터저장소_스키마.md` 4장 형식으로 변환한 산출물이다.
원본은 수정하지 않았으며, `scripts/normalize_corpus.py`를 다시 실행하면 전체가 재생성된다.

- **202개 파일, 문단 20,359개** = 정본 102파일 9,885(감사기준 3,630 + 회계기준 6,255)
  + **BC·IE 갈래 100파일 10,474**(결론도출근거 8,721 + 적용사례 1,753) — 전 문단 ID 유일성 검증 완료
- 실무지침 9건(`guidelines_md/`)과 합치면 규약이 말하는 전체 코퍼스가 된다.
  청크 ID: `KSA::<번호>::<문단>` / `KIFRS::<번호>::<문단>` / `GUIDE::<번호>::<문단>`

## 파일 명명과 standard_no

| 묶음 | 파일명 | standard_no 예 |
|---|---|---|
| 감사기준서 200~720, 1100, 1200 | `ksa_200.md` … | `"200"`, `"1100"` |
| 품질관리기준서 1 (2018 제정) | `ksa_isqm-1.md` | `"ISQM-1"` |
| 인증업무개념체계 (2022 개정) | `ksa_frmk-1.md` | `"FRMK-1"` |
| 기타인증업무기준 (2022 개정) | `ksa_assr-3000.md` | `"ASSR-3000"` |
| K-IFRS 기준서·해석서 | `kifrs_1115.md`, `kifrs_2032.md` | `"1115"`, `"2032"` |
| 재무보고를 위한 개념체계 | `kifrs_cf.md` | `"CF"` |
| 경영진설명서 개념체계 (실무서 1) | `kifrs_mc.md` | `"MC"` |
| 중요성 실무서 (실무서 2) | `kifrs_ps2.md` | `"PS2"` |

품질관리기준서 1과 인증업무개념체계는 둘 다 원본 번호가 "1"이라 충돌하므로
기존 파싱 프로젝트의 식별자(ISQM-1/FRMK-1/ASSR-3000)를 standard_no로 유지했다.

## 수록 범위 결정

1. **K-IFRS 정본(authority 1) 구성요소** — 본문·용어의 정의·부록 적용지침(B·C 등)·경과규정·한국 문단 —
   는 정본 파일(`kifrs_NNNN.md`)에 수록한다. **결론도출근거(BC)와 적용사례(IE)는 2026-07-12부터
   별도 갈래 파일(`kifrs_NNNN_bc.md`/`_ie.md`, 규약 4.3-5)로 수록**한다 — 기준서의 일부는 아니지만
   감사 논리구축에 쓰이는 해설·예시 자료(para_type "결론도출근거"/"적용사례", 검색은 옵트인
   include_bc/include_examples). 정본 파일에는 절대 섞지 않는다(파일 분리가 C-07 재발 방지의
   기계적 증명 — LEDGER D-22). 번호가 재시작하는 IE는 `IE사례N-{원번호}` 식 합성 ID를 쓴다
   (초판의 "ID 충돌로 제외" 결정을 합성 문법으로 해소 — 원번호는 항상 보존).
2. **개념체계 3종은 결론도출근거만 제외하고 전부 수록** (문서 자체가 참고문헌 성격).
   경영진설명서·실무서2는 원본 md에서 본문이 `ie`로 오태깅되어 있어 component 태그 대신 이 규칙을 쓴다.
3. **`00_전문.md`(회계감사기준 목차)는 변환하지 않음** — 목차는 규약 4.2 제거 대상.
4. **감사기준 보론(번호 없는 부록)은 규약 4.3의 가상 번호로 성격별 분리 수록** —
   - 예시 보고서·예시 문서 한 건 = `부록-사례N`: 700(1~4), 705(1~5), 720(1~7), 1200(1~10),
     710(1~4), 570(1~3), 1100(1~3, `####` 사례 제목이 절단선), 510(1~2), 580 보론2(진술서 예시=사례1)
   - 다른 기준서의 요구사항 위치를 나열하는 목록형 보론 = `참조N`(para_type "참조", 검색 기본 포함):
     706 보론1·2, 580 보론1, 230, 260 보론1
   - 부록 서두·목록형 부록·고려사항 목록 = `부록N`: 240(1~3), 600(보론 1~5 서두),
     300(보론 서두+하위 절 4개 = 1~5), 620, 260 보론2, 706 보론3·4(예시 보고서 통짜),
     210(보론1 계약서 예시, 보론2 서두), 315(서두 1·2·4·5·6 — 보론3은 서두 없음), 530(1~4),
     FRMK-1(보론1 도표 전체 + 보론 2~4 무번호 서두 = 1~4)
   - 원본이 자체 번호를 부여한 보론 문단(`부록-N`, `보론N-M`)은 그대로 유지하며, 위 가상
     번호와 공존한다 (예: 210 부록1·부록2 서두 뒤에 부록-1~5, 600 부록4 서두 뒤에 부록-51~).
   - 540(제목 이어행)과 ASSR-3000(보론 참조·제목 행)은 조각 대신 절 제목에 흡수했다.
   예시문 내부의 짧은 소제목처럼 보이는 행은 예시문 분절을 피하기 위해 본문에 흡수했다.
   조서 라우팅 표(저장소 밖, 규약 2.3-3) 작성 시 옵트인 예정 행: **조서 2110(감사계획) →
   감사기준 300 부록 포함**.
5. **실무서 2(PS2)의 사례 상자와 부록 발췌를 분리 수록** — 표 형식 사례 상자 20개를
   `사례A`~`사례T`로 절단해 호스트 문단 15개를 정상화했고, 문단 89 뒤에 병합돼 있던
   부록 "재무보고를 위한 개념체계와 기업회계기준서 참조"(9,525자)는 원천 굵은 글씨 제목
   ("…에서 발췌") 경계로 `참조1`~`참조4`(각 6,000자 미만)로 절단했다. PS2 한정 결정이며,
   발췌 안의 개별 상호참조(cross_refs) 필드 승격은 백로그로 남긴다.
6. **para_type 유도 규칙 (확정판)** — 예시류(검색 기본 정책에서 후순위·제외, 문안 작성 의도
   질의와 조서 라우팅 표의 명시 포함에서만 개방): `부록N`·`부록-사례N`·`사례X`.
   `참조N`은 para_type "참조"로 검색 기본 포함. 그 외는 기존 규칙(A접두=적용지침,
   굵은체=요구사항, `부록-N` 원본 번호 문단=부록 등) 유지.
7. **부록 A "용어의 정의"는 용어집 갈래(규약 4.3-3, v2.1)로 행 단위 수록** — 신기준 계열
   16개(1101~1117)의 부록 A 표와 감사기준 1200 보론 1의 용어 표(65용어)를 표의 행 = 용어
   하나로 절단했다. ID는 `정의-{정규화된 용어명}`(예: `KIFRS::1115::정의-수행의무`,
   `KSA::1200::정의-지배력` 아닌 `KSA::1200::정의-감사위험` 등), para_type "정의"(검색 기본
   포함), 본문은 `{용어}: (정의문)` 형태, has_table=false(행 하나의 산문화).
   - **용어명 정규화 규칙**: 원문 표기에서 각주 마커(`[^N]`·`한N)`)와 굵은 글씨 마커를 제거한 뒤
     **공백만 제거**하고 괄호·가운뎃점·하이픈은 보존한다(예: `(감사증거의) 적합성` →
     `정의-(감사증거의)적합성`, `12개월 기대신용손실` → `정의-12개월기대신용손실`).
     후속 용어 사전 구축기가 같은 규칙을 쓴다.
   - 부록 A 안의 "다음 용어는 타 기준서에서 정의한다" 안내문 + 용어 나열 블록은 `참조1`로
     수록했다(1107, 1109, 1110, 1111, 1112 — para_type "참조"). 1116은 같은 성격의 블록이
     정의문을 가진 표라서 하위 절 "다른 기준서에서 정의하고 있으며 …" 아래 정의 조각으로 수록.
   - 구기준(IAS 1001~1041)은 정의가 본문 번호 문단(예: 1002::6)이라 대상이 아니고, 해석서·
     개념체계·실무서와 감사기준(1200 제외)도 원문 구조상 부록 A 정의부가 없다.
   - ksa_1200의 기존 `부록1`(통짜 표 9,036자, 6,000자 초과 허용 목록에 있던 조각)은 이
     분해로 **소멸**했다. 정의 조각의 절 제목은 원본 유래 "보론 1 용어의 정의"를 유지한다.

## 원본 데이터 결함과 보정

결함 보정 기록 전수(C-01~C-13: 문단번호 오류, ISQM-1 번호 소실, IE 유출, 부록 행머리
병합, 용어정의 병합 등)는 [`docs/LEDGER.md`](../docs/LEDGER.md)로 이관했다.

## 형식 특징 (파서 구현 시 참고)

- frontmatter: `source_type`(감사기준/회계기준), `standard_no`(따옴표 문자열), `standard_title`, `origin`(원본 경로 — 파서는 무시 가능)
- 절 제목은 `## 상위 > 하위` 한 줄로 계층을 합성했다 → 그대로 `section_path`로 쓰면 된다
- 문단 행머리 정규식(변환기의 검증과 동일):
  `^(부록-사례\d+|부록-?[0-9A-Za-z()]+|보론\d*-\d+|사례[A-Z]|참조\d+|정의-[가-힣A-Za-z0-9()·-]+|한?[A-Z]{0,4}\.?\d[0-9A-Za-z.-]*)\.\s`
  (`부록-사례N`은 예시 보고서 사례, `사례[A-Z]`는 PS2 전용, `참조N`은 PS2 부록 발췌와
  부록 A의 타 기준서 참조 블록, `정의-{용어}`는 용어집 조각(수록 범위 결정 7),
  `\.?\d`는 1109의 `BA.1` 같은 '영문자+점+숫자' 계열)
- 굵은 글씨(`**…**`)는 K-IFRS 의무규정 문단 표시로 보존됨
- 하위 항목 `(a)`, `⑴`, `•` 행과 표(`|`)는 해당 문단에 속하는 이어지는 행이다.
  행머리 번호로 오인될 수 있는 이어지는 행은 변환기가 탭으로 들여썼다
- 각주는 원본 전처리 단계에서 이미 분리되어 md에 없음 (K-IFRS 각주는 `_IFRS_parsing` DB의 footnotes 테이블 참조)
- **6,000자 초과 문단 1건** (임베딩 입력 한도 대비 — 규약 3.6의 분할 표시 대상):
  ksa_240 부록1(6,024자 — 부정위험요소 예시 목록). 검증기가 이 1건 외의 초과를 실패로 처리한다.
- **상설 구조 검사**: 정규 번호 문단 블록이 '보론' 포함 절 제목(`##`)을 가로지르면 변환기
  검증이 실패한다 — 크기와 무관하게 보론 병합의 재발을 차단하는 검사다.
- ISQM-1 재생성은 원본 DOCX가 없어도 가능 — 파싱 결과가 `scripts/isqm1_rows.json`에 동봉되어 있다.
