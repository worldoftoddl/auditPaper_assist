# corpus_md — 규약 형식 통일 코퍼스 (감사기준 + 회계기준)

`auditstandard_md/`(감사기준 40개)와 `ifrs_md/`+`Conceptual_framework_md/`(K-IFRS 63개)를
`guidelines_raw/벡터저장소_스키마_및_마크다운_작성규약.md` 4장 형식으로 변환한 산출물이다.
원본은 수정하지 않았으며, `scripts/normalize_corpus.py`를 다시 실행하면 전체가 재생성된다.

- **102개 파일, 문단 9,568개** (감사기준 3,533 + 회계기준 6,035) — 전 문단 ID 유일성 검증 완료
- 실무지침 9건(`guidelines_md/`)과 합치면 규약이 말하는 전체 코퍼스가 된다.
  청크 ID: `KSA::<번호>::<문단>` / `KIFRS::<번호>::<문단>` / `GUIDE::<번호>::<문단>`

## 파일 명명과 standard_no

| 묶음 | 파일명 | standard_no 예 |
|---|---|---|
| 감사기준서 200~720, 1100, 1200 | `ksa_200.md` … | `"200"`, `"1100"` |
| 품질관리기준서 1 (2018 제정) | `ksa_isqm-1.md` | `"ISQM-1"` |
| 인증업무개념체계 (2022 개정) | `ksa_frmk-1.md` | `"FRMK-1"` |
| 기타인증업무기준 (2022 개정) | `ksa_assr-3000.md` | `"ASSR-3000"` |
| K-IFRS 기준서·해석서 | `kifrs_1115.md`, `kifrs_2032.md` | `"1115"`, `"2032"` |
| 재무보고를 위한 개념체계 | `kifrs_cf.md` | `"CF"` |
| 경영진설명서 개념체계 (실무서 1) | `kifrs_mc.md` | `"MC"` |

품질관리기준서 1과 인증업무개념체계는 둘 다 원본 번호가 "1"이라 충돌하므로
기존 파싱 프로젝트의 식별자(ISQM-1/FRMK-1/ASSR-3000)를 standard_no로 유지했다.

## 수록 범위 결정

1. **K-IFRS는 정본(authority 1) 구성요소만 수록** — 본문·용어의 정의·부록 적용지침(B·C 등)·경과규정·한국 문단.
   **결론도출근거(BC)와 적용사례(IE)는 제외**했다. 규약의 para_type 체계(요구사항/적용지침/본문/부록)와
   문단 수 추정(1~2만)이 정본 기준이고, 적용사례는 절마다 번호가 1부터 재시작해 ID가 충돌하기 때문.
   필요해지면 `IE-사례N-M` 식 접두를 도입해 변환기를 확장하면 된다.
2. **개념체계 3종은 결론도출근거만 제외하고 전부 수록** (문서 자체가 참고문헌 성격).
   경영진설명서·실무서2는 원본 md에서 본문이 `ie`로 오태깅되어 있어 component 태그 대신 이 규칙을 쓴다.
3. **`00_전문.md`(회계감사기준 목차)는 변환하지 않음** — 목차는 규약 4.2 제거 대상.
4. **감사기준 예시문(번호 없는 부록)은 규약 4.3의 가상 번호로 분리 수록** — 9개 파일에서
   부록 서두·목록형 부록은 `부록N`, 예시 보고서 한 건은 `부록-사례N`으로 절단했다
   (700: 사례 1~4, 705: 1~5, 720: 1~7, 1200: 1~10, 710: 1~4, 570: 1~3, 1100: 1~3,
   240: 부록 1~3, 600: 보론 1~5 서두 = 부록1~5, 확인서·지침서한 항목은 원본 유래 `부록-51`~ 유지).
   1100은 사례가 `####` 절 제목으로 구분되어 있어 제목 행을 절단선으로 썼다.
   예시문 내부의 짧은 소제목처럼 보이는 행은 예시문 분절을 피하기 위해 본문에 흡수했다.
5. **실무서 2(PS2)의 사례 상자와 부록 발췌를 분리 수록** — 표 형식 사례 상자 20개를
   `사례A`~`사례T`로 절단해 호스트 문단 15개를 정상화했고, 문단 89 뒤에 병합돼 있던
   부록 "재무보고를 위한 개념체계와 기업회계기준서 참조"(9,525자)는 원천 굵은 글씨 제목
   ("…에서 발췌") 경계로 `참조1`~`참조4`(각 6,000자 미만)로 절단했다. PS2 한정 결정이며,
   발췌 안의 개별 상호참조(cross_refs) 필드 승격은 백로그로 남긴다.
6. **para_type 유도 규칙 (확정판)** — 예시류(검색 기본 정책에서 후순위·제외, 문안 작성 의도
   질의와 조서 라우팅 표의 명시 포함에서만 개방): `부록N`·`부록-사례N`·`사례X`.
   `참조N`은 para_type "참조"로 검색 기본 포함. 그 외는 기존 규칙(A접두=적용지침,
   굵은체=요구사항, `부록-N` 원본 번호 문단=부록 등) 유지.

## 원본 데이터 결함과 보정 (원문 훼손 아님 — 추출 오류의 복원)

1. **감사기준 4개 파일의 문단번호 국소 오류** — 원본 DOCX 자동번호 재시작으로 추출 번호가 어긋난 곳을
   적용자료 절 제목의 "(문단 N 참조)" 상호참조와 국제감사기준 대조로 복원했다
   (`normalize_corpus.py`의 `ISA_CORRECTIONS`, idx 기준):
   - ISA-250: 요구사항 두 번째 12→**15**
   - ISA-260: 유의적 발견사항 5→**16**, 독립성 6→**17**
   - ISA-300: 7(중복)→**8**, 8→**9**, 문서화 9→**12**, 초도감사 10→**13**
   - ISA-701: 두 번째 문단4부터 전부 +1 (4→**5** … 17→**18**)
2. **ISQM-1(품질관리기준서 1)은 md에서 문단번호가 전부 소실** — 원본 DOCX가 2열 표(열0=문단번호)
   구조라 기존 파서가 번호를 놓쳤다. 변환기가 DOCX 표를 직접 읽어 1~59, 한4-1 등, A1~A75를 복원했다.
   원본 DOCX 경로: `_AuditStandard_parsing/raw/3. 품질관리기준서1(2018년 제정)_국어전문.docx`
3. **FRMK-1(인증업무개념체계)** — 절 제목이 문서 머리에 목차 형태로만 있어 `range` 속성으로
   문단↔절을 재결합했다. 보론 1~4는 자체 번호가 1부터 재시작하므로 `보론N-M` 접두를 부여했다.
4. **ASSR-3000(기타인증업무기준)** — md에 절 제목이 전혀 없어 짧은 독립 행 휴리스틱으로 복원했다.
   보론 문단은 `보론-M` 접두. 문단 46~49는 원문이 제한적/합리적 확신 2열 대비표(46L/46R 형식)라
   표 통째로 인접 문단에 붙어 있다(개별 ID 없음 — 알려진 한계).
5. **K-IFRS 제1033호(주당이익)** — 원본 md에 기준서 전체가 두 번 반복 수록되어 있어 첫 번째 본만 취했다.
6. **K-IFRS 제1007호(현금흐름표)** — 예시 성격의 부록 A·B·C가 authority 1로 오태깅되어 있어 명시적으로 제외했다.
7. **결론도출근거(`## 결론도출근거`)·적용사례(`## 적용사례`) 이후의 component 태그는 전부 무시** —
   그 뒤의 `transition/main/definitions/ag authority 1` 태그는 모두 BC·IE 내부 절의 오태깅으로 확인됐다.
   (제1109호에서 적용사례 구역 한가운데의 `definitions authority 1` 오태깅이 IE 콘텐츠 33,711자를
   유출시킨 것이 계기.) 비BC·IE 문단이 이 규칙으로 제외되면 변환기가 경고를 출력한다.
8. **개념체계 서문(SP1.1~SP1.5)** 은 원본에 para 주석이 없어 행머리 패턴으로 직접 절단했다.
9. **K-IFRS 부록 정본 문단 166개의 행머리 복원** — 16개 파일의 부록 구역(1101 D1~D36·E계열,
   1036·1021·1033·2102 등의 A계열, 1109 BA.1~BA.8, 1111 B33CA·C1AA·C1AB, 1116 C20BA~C20BC,
   1115 한C1.1, 1117 한C2.1, 1039 AG99BA 등)이 원본 md에 para 주석 없이 무마침표 행머리
   (`A1<TAB>본문`)로만 남아 있어 직전 문단에 병합되어 있었다. 변환기가 행머리 패턴 + 계열 연속성
   (직전 문단과 같은 영문자 계열이거나 계열 시작 X1)으로 승격해 복원했다.
10. **제1109호(금융상품) B7.2.5~B7.2.42 부재는 원본 자체의 상태** — 국제기준(IFRS 9) 구조상
    존재해야 할 경과규정 적용지침 문단들이 원본 DOCX(2025 개정 통합본)에도 없음을 확인했다
    (B7.2.4까지만 존재). 추출 손실이 아니므로 복원 대상이 아니다.
11. **비정본 오수록 50개 문단 제거** — 제1034호 부록 A·B(A1~A2, B1~B36)는 원문이
    "이 기준서의 일부를 구성하는 것은 아니다"라고 선언한 적용사례 첨부물인데 authority 1로
    오태깅되어 있었고, 제1117호 IE186~IE199는 `## 경과규정` 오태깅 아래 놓인 적용사례 문단이었다.
    모두 수록 범위 결정 1(BC/IE 제외)에 따라 제외했다.
12. **의결 문구 제거** — "회계기준위원회 위원: …", "…의 개정(날짜)은 회계기준위원회 위원 N인
    …의결하였다" 형태의 의결 사실 기재는 정본 문단이 아니므로 전 파일에서 제거했다.
    원본의 `authority_declaration: authoritative/non-authoritative` 주석도 component 태그와
    동급의 포함/제외 신호로 반영한다.

## 형식 특징 (파서 구현 시 참고)

- frontmatter: `source_type`(감사기준/회계기준), `standard_no`(따옴표 문자열), `standard_title`, `origin`(원본 경로 — 파서는 무시 가능)
- 절 제목은 `## 상위 > 하위` 한 줄로 계층을 합성했다 → 그대로 `section_path`로 쓰면 된다
- 문단 행머리 정규식(변환기의 검증과 동일):
  `^(부록-사례\d+|부록-?[0-9A-Za-z()]+|보론\d*-\d+|사례[A-Z]|참조\d+|한?[A-Z]{0,4}\.?\d[0-9A-Za-z.-]*)\.\s`
  (`부록-사례N`은 예시 보고서 사례, `사례[A-Z]`·`참조N`은 PS2 전용, `\.?\d`는 1109의 `BA.1` 같은
  '영문자+점+숫자' 계열)
- 굵은 글씨(`**…**`)는 K-IFRS 의무규정 문단 표시로 보존됨
- 하위 항목 `(a)`, `⑴`, `•` 행과 표(`|`)는 해당 문단에 속하는 이어지는 행이다.
  행머리 번호로 오인될 수 있는 이어지는 행은 변환기가 탭으로 들여썼다
- 각주는 원본 전처리 단계에서 이미 분리되어 md에 없음 (K-IFRS 각주는 `_IFRS_parsing` DB의 footnotes 테이블 참조)
- **6,000자 초과 문단 2건** (임베딩 입력 한도 대비 — 적재기의 분할 정책 대상):
  ksa_1200 부록1(9,036자 — 용어 정의 표 66행, 규약 4.2.4에 따라 통짜 유지),
  ksa_240 부록1(6,024자 — 부정위험요소 예시 목록). 검증기가 이 2건 외의 초과를 실패로 처리한다.
- ISQM-1 재생성은 원본 DOCX가 없어도 가능 — 파싱 결과가 `scripts/isqm1_rows.json`에 동봉되어 있다.
